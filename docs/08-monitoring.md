# Network Monitoring

## 🎯 Objective
Monitor network traffic, device activity, and interface usage on MikroTik.

---

## 🔍 Why Monitoring is Important
- Detect abnormal traffic
- Identify bandwidth usage
- Troubleshoot network issues
- Monitor device performance

---

## 🧰 Tools Used

### 1. Torch (Built-in MikroTik Tool)
- Real-time traffic monitoring
- Shows:
  - Source IP
  - Destination IP
  - Port
  - Protocol

📌 Location:
Tools → Torch

---

### 2. Interface Traffic
- View bandwidth usage per interface
- Useful for:
  - Checking WAN usage
  - Monitoring LAN activity

📌 Location:
Interfaces → Select Interface → Traffic Tab

---

### 3. Traffic Graph
- Provides historical bandwidth data
- Accessible via web browser

Example:
http://192.168.10.1/graphs/

---

### 4. The Dude (Optional)
- MikroTik network monitoring tool
- Supports topology view and alerts

---

## ⚙️ Basic Monitoring Setup

### Enable Graphing
- Interfaces
- Queues
- System resources

---

## 📊 Expected Result
- Real-time traffic visibility
- Historical bandwidth graphs
- Better troubleshooting capability
